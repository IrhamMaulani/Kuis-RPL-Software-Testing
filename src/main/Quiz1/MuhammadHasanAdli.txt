Soal
1.	Dewasa ini, pembuatan testing pada perangkat lunak umum dilakukan pada saat tahap implementasi yang dilakukan oleh software engineer. Berikan opini kalian apakah software engineer pada tahap implementasi wajib melakukan test atau tidak?
2.	Unit test dan integration test merupakan test yang dilakukan pada tahap implementasi. berikan pendapat kalian jika dua test ini tidak dilakukan beserta dampak yang akan terjadi pada perangkat lunak
3.	Berikan pendapat kalian tentang pekerjaan QA Engineer sedangkan disisi lain Software Engineer telah melakukan testing
4.	Terdapat banyak test pada perangkat lunak, mulai dari test business logic, UI, UX, Performa, sampai keamanan. Namun tidak semuanya dapat dilakukan karena keterbatasan waktu dan mahalnya biaya yang harus dikeluarkan. Berikan pendapat kalian dampak apa yang terjadi jika pengujian diatas tidak diberikan pada perangkat lunak
5.	Berikan pendapat kalian tentang testing perangkat lunak secara manual? Apakah di saat banyaknya testing secara otomatisasi dilakukan testing secara manual perlu dilakukan?
Jawab
1.	Menurut saya software testing wajib untuk dilakukan untuk memeriksa dan mengetahui apakah sebuah perangkat lunak sudah berjalan dengan baik atau tidak, dan berfungsi untuk mengetahui kualitas perangkat lunak tersebut.
https://glints.com/id/lowongan/software-testing-adalah/#.YmZZytpBxnI
2.	Berdasarkan artikel yang saya baca di https://glints.com/id/lowongan/unit-testing-adalah/#.YmZa0dpBxnI unit test dan integration test merupakan tahap yang wajib untuk dilakukan karena berfungsi untuk memastikan bahwa setiap unit kode software sudah bisa bekerja sesuai harapan. Jika 2 tahap tersebut tidak dilakukan takutnya perangkat lunak yang kita buat tidak berjalan sebagaimana mestinya, jadi menurut saya 2 tahap itu wajib untuk dilakukan supaya perangkat lunak yang kita buat berjalan sesuai harapan yang kita inginkan.
3.	Berdasarkan artikel yang saya baca di https://glints.com/id/lowongan/qa-quality-assurance-engineer/#.YmZcPNpBxnI tugas seorang QA Engineer dan software tester tidak jauh beda bahkan hamper sama, namun pasti ada perbedaan pada 2 pekerjaan tersebut, seorang software tester biasanya bertugas mencari bug dan men cek apakah ada eror dalam suatu aplikasi, sedangkan QA Engineer juga bertugas mencari bug, namun mereka juga punya tugas untuk memastikan customer experience.
4.	Menurut saya semua tahap pengujian tersebut penitng untuk dilakukan, jika kita tidak melakukan test business logic maka kita tidak akan tau apakah sebuah fungsi sudah berjalan dengan benar, UI dan UX berguna untuk antarmuka pada perangkat lunak jika tidak dilakukan pengujian maka kita tidak akan tau apakah interface tersebut sudah sesuai dengan yang kita harapkan, dan jika kita tidak melakukan test pada performa dan keamannya maka kita tidak akan tahu Batasan dari perangkat lunak yang kita buat dan kita tidak tahu seberapa aman perangkat lunak tersebut untuk digunakan.
5.	Berdasarkan artikel yang saya baca di https://medium.com/skyshidigital/perbedaan-manual-testing-dan-automated-testing-9d13373a36e testing manual adalah sebuah metode pengujian untuk mencari cacat atau bug yang ada pada perangkat lunak. Manual testing bisa saja dilakukan jika automated testing tidak dapat dijalankan.
