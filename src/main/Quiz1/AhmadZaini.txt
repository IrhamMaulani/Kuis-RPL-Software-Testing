4. Berdasarkan situs https://testfort.com/blog/business-logic-in-qa-and-its-role-in-the-development-of-perfect-software , Business logic merupakan sistem untuk memandu bagaimana objek bisnis bekerja satu sama lain
sedangkan untuk UI dan UX berdasarkan pada situs " https://www.testgorilla.com/test-library/role-specific-skills-tests/ux-ui-design-test/ " ,
Pengujian UX berfokus apakah pengguna menikmati pengalaman serta kemudahan dari titik A ke B tanpa ada kendala
Pengujian UI berfokus memastikan pada layar visual pengguna telah sesuai dengan jalur A dan jalur B sehingga pengguna dengan mudah berinteraksi ke perangkat lunak
Jadi pendapat saya, test business logic jika tidak dilakukan maka pada perangkat lunak tidak dapat memandu fungsi dari sistem, UI dan UX yang merupakan antarmuka pada perangkat lunak tidak akan tau apakah pengguna dapat mengakses fitur dengan mudah atau visual yang diberikan tidak sesuai,untuk performa dan keamanan dapat disimpulkan jika performa dalam suatu perangkat lunak tidak diuji maka kita tidak akan tau apakah ketika banyak pengguna mengakses akan membuat perangkat lunak menjadi malfungsi atau tidak,sedangkan pada sisi keamanan sangat penting untuk diuji karena mencegah terjadinya kebocoran data dan seharusnya untuk performa dan keamanan ini perlu di update dan maintenance setiap bulannya.

5. Berdasarkan yang saya baca di situs https://medium.com/dot-intern/manual-testing-vs-automated-testing-9244aaed1ed5
Manual testing adalah pengujian oleh QA untuk memeriksa semua fitur pada software, untuk memastikan apakah ada bug atau tidak dalam software.
Automation testing berfungsi untuk membandingkan hasil yang diharapkan dengan hasil sebenarnya,dengan automated testing pengujian dapat dilakukan secara berulang
Manual testing diperlukan jika automated testing tidak dapat dijalankan. Adapun beberapa kelebihan manual testing ialah
- Mendapatkan visual feedback
- Bisa mendapatkan feedback dari orang secara langsung sehingga mengetahui user suka atau tidak
- lebih murah dalam proyek jangka pendek,maka manual testing tidak perlu menggunakan tools ataupun software yang mahal
- Bisa menghemat waktu
